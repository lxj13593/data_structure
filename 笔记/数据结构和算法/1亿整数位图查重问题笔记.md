# 1 亿整数查重问题：位图与 2 位状态笔记

## 1. 题目

有 1 亿个非负整数，每个整数最大值不超过 1 亿，内存限制为 100 MB。

要求解决：

1. 哪些元素重复出现；
2. 按输入顺序，谁是第一个重复出现的元素；
3. 按输入顺序，谁是第一个只出现一次的元素；
4. 理解用 1 个 bit 和 2 个 bit 保存数据状态的方法。

---

## 2. 为什么不能直接保存全部整数

一个 `int` 通常占 4 字节：

```text
1 亿 × 4 字节 = 4 亿字节 ≈ 400 MB
```

超过 100 MB，因此不能直接把全部整数保存在普通数组、`set` 或 `unordered_set` 中。

---

# 一、1 个 bit 保存状态

## 3. 状态含义

为每个可能的数字分配 1 个 bit：

```text
0：没有出现
1：已经出现
```

数字范围为：

```text
0 ～ 100000000
```

一共需要：

```text
100000001 个 bit
```

内存约为：

```text
100000001 ÷ 8 ≈ 12500001 字节 ≈ 11.92 MiB
```

符合 100 MB 限制。

---

## 4. 数字如何映射到位图

如果使用 32 位无符号整数数组，每个数组元素能记录 32 个数字。

对于数字 `num`：

```text
数组下标 = num / 32
位下标   = num % 32
```

判断是否出现：

```cpp
(bitmap[index] & (1u << offset)) != 0
```

设置为已经出现：

```cpp
bitmap[index] |= (1u << offset);
```

---

## 5. 查找第一个重复数字

“第一个重复”指：

> 从左到右扫描时，最先遇到的第二次出现的数字。

例如：

```text
3 5 7 5 3
```

扫描到第二个 `5` 时，发现对应 bit 已经是 1，因此第一个重复数字是 `5`。

处理过程：

```text
bit 为 0：设置为 1，继续
bit 为 1：当前数字就是第一个重复数字，结束
```

时间复杂度：

```text
O(n)
```

空间约为：

```text
12.5 MB
```

---

## 6. 1 个 bit 的限制

例如输入：

```text
1 2 1 1 2
```

第二次、第三次遇到 `1` 时，对应 bit 都是 1，因此可能输出：

```text
1 1 2
```

这表示输出每一次重复事件，而不是每个重复数字只输出一次。

因为 1 个 bit 只能区分：

```text
没出现
出现过
```

不能区分：

```text
出现 1 次
出现 2 次
出现 3 次以上
```

所以：

- 找第一个重复：1 bit 足够；
- 每个重复数字只输出一次：需要更多状态。

---

# 二、2 个 bit 保存状态

## 7. 三种有效状态

两个 bit 可以表示四种组合，通常使用其中三种：

```text
00：没有出现
01：出现 1 次
10：出现 2 次或更多
11：保留不用
```

状态变化：

```text
00 → 01 → 10
```

含义：

```text
第一次出现：00 → 01
第二次出现：01 → 10
第三次及以后：10 → 10
```

---

## 8. 2 位状态需要多少内存

每个数字使用 2 bit：

```text
100000001 × 2 bit
≈ 200000002 bit
≈ 25000001 字节
≈ 23.84 MiB
```

仍然小于 100 MB。

如果使用 32 位整数数组，一个整数可以记录：

```text
32 ÷ 2 = 16 个数字
```

对于数字 `num`：

```text
数组下标 = num / 16
组内位置 = num % 16
位偏移   = (num % 16) × 2
```

---

## 9. 查找所有重复数字，并且每个只输出一次

从头扫描数据：

```text
状态 00：
改为 01，不输出

状态 01：
改为 10，并输出
这是该数字第一次被确认重复

状态 10：
保持 10，不再输出
```

例如：

```text
输入：1 2 1 1 2 3 3
```

处理结果：

```text
1：00 → 01
2：00 → 01
1：01 → 10，输出 1
1：10 → 10，不输出
2：01 → 10，输出 2
3：00 → 01
3：01 → 10，输出 3
```

最终输出：

```text
1 2 3
```

---

## 10. 查找第一个重复数字

2 位状态也可以做：

```text
遇到 00：改成 01
遇到 01：说明这是第二次出现，立即返回
遇到 10：说明以前已经重复
```

但只为了找第一个重复，1 位位图已经足够。

---

## 11. 查找第一个不重复数字

“第一个不重复”指：

> 按原始输入顺序，第一个总共只出现一次的数字。

例如：

```text
4 2 4 3 2 5
```

出现次数：

```text
4：2 次
2：2 次
3：1 次
5：1 次
```

按输入顺序，第一个只出现一次的是：

```text
3
```

需要两遍扫描。

第一遍：

```text
统计每个数字的状态
00 → 01 → 10
```

第二遍：

```text
再次按原始输入顺序读取
找到第一个状态为 01 的数字
```

它就是第一个不重复数字。

---

## 12. 为什么需要第二遍

位图只按数字大小保存状态，不保存原始输入顺序。

例如：

```text
输入：8 3 8 5
```

位图只能告诉我们：

```text
3 出现一次
5 出现一次
8 重复
```

但不能单独告诉我们谁在原输入中更早出现。

所以必须：

```text
第一遍统计次数状态
第二遍按原输入顺序寻找第一个状态为 01 的数字
```

如果数据来自文件，可以重新读取文件。

如果数据只是一遍数据流，且不能再次读取，也不能把 1 亿个整数全部放入内存，那么仅靠位图无法恢复原始顺序，需要借助外部文件或其他存储方式。

---

# 三、两个 1 位位图也可以表示 2 位状态

## 13. 两张位图的含义

可以建立：

```text
bitmap_once：是否至少出现过一次
bitmap_repeat：是否已经重复
```

状态：

| once | repeat | 含义 |
|---:|---:|---|
| 0 | 0 | 没出现 |
| 1 | 0 | 出现一次 |
| 1 | 1 | 出现两次或更多 |

处理逻辑：

```text
once = 0：
设置 once = 1

once = 1 且 repeat = 0：
设置 repeat = 1，并输出重复数字

repeat = 1：
已经确认重复，不再重复输出
```

两张位图总内存约为：

```text
12.5 MB × 2 = 25 MB
```

和单个 2 位状态表基本相同。

---

# 四、不同问题分别需要什么

| 问题 | 需要的状态 | 扫描次数 |
|---|---:|---:|
| 判断数字是否出现过 | 1 bit | 1 次 |
| 找第一个重复数字 | 1 bit | 最多 1 次 |
| 输出每一次重复事件 | 1 bit | 1 次 |
| 每个重复数字只输出一次 | 2 bit 或两个位图 | 1 次 |
| 找第一个不重复数字 | 2 bit 或两个位图 | 2 次 |
| 统计准确出现次数 | 2 bit 不够 | 需要计数数组或其他结构 |

---

# 五、复杂度

## 时间复杂度

单次位图查询、设置：

```text
O(1)
```

处理全部 1 亿个整数：

```text
O(n)
```

找第一个不重复需要两遍：

```text
O(2n)
```

忽略常数后仍然是：

```text
O(n)
```

## 空间复杂度

```text
1 位位图：约 12.5 MB
2 位状态：约 25 MB
两个 1 位位图：约 25 MB
```

都符合 100 MB 限制。

---

# 六、注意事项

## 1. 最大值是否包含

如果最大值可能等于 `100000000`，需要记录：

```text
0 ～ 100000000
```

因此是 `100000001` 个状态。

数组大小通常写成：

```text
maxNumber / 每组可记录数量 + 1
```

避免最大值越界。

## 2. 数字必须是非负数

位图下标直接由数字计算，因此需要：

```text
0 <= num <= 100000000
```

如果有负数，需要做偏移映射，或者使用其他数据结构。

## 3. 明确“输出重复”的含义

每次重复都输出：

```text
输入：1 1 1
输出：1 1
```

每个重复数字只输出一次：

```text
输入：1 1 1
输出：1
```

后者需要 2 位状态或额外位图。

---

# 七、核心记忆

```text
1 亿个 int 约占 400 MB，超过 100 MB。

1 位位图约占 12.5 MB：
0 表示没出现，1 表示出现过。

找第一个重复：
第一次遇到对应 bit 已经为 1 的数字即可。

2 位状态约占 25 MB：
00 没出现
01 出现一次
10 出现两次及以上

找所有重复且每个只输出一次：
状态从 01 变成 10 时输出。

找第一个不重复：
第一遍统计状态，
第二遍按原输入顺序找第一个状态为 01 的数字。

单次位图操作是 O(1)，
处理全部数据的整体复杂度仍然是 O(n)。
```
